I love Groovy, but I was loving a lie... okay, all kidding aside, I had some assumptions about how Groovy behaved, that I have recently found to be wrong. Truly, this is no fault of Groovy, but it is easy to fall into this assumption if you are not careful. Consider the following set of assertions: 
final long foo = 123
assert foo == 123

foo = 246
assert foo == 246

Map<Long,Date> map = [:] as Map<Long,Date>
map.foo = 'bar'
assert map['foo'] == 'bar'

Map<Long,Date> map2 = new HashMap<Long,Date>()
map2.foo = 'bar'
assert map2['foo'] == 'bar'

All four will assert true. Now, go back and look at the code with a Java-eye... see what I mean? Coming from a Java background I had always assumed that Groovy would at least honor these constructs if you used them, though it would not require them. To me this is worse than not allowing them at all since it has given me a false sense of security.

Live and learn... and remember the rule of assumptions.
